# 项目宪法 (Project Constitution)

## Python 代码注释规则

本工程内所有 Python 代码（新增或修改）**必须添加中文注释**，原因：项目所有者是 Python 初学者，需要通过注释理解代码。

### 具体要求

1. **每个函数/方法**：在 `def` 下方加 docstring，说明：
   - 这个函数做什么
   - 参数含义
   - 返回值含义

2. **每个类**：在 `class` 下方加 docstring，说明这个类的用途。

3. **关键代码行**：对以下情况加行内或上方注释：
   - 复杂的列表/字典推导式
   - 不直观的算法步骤
   - 第三方库的非标准用法
   - 正则表达式
   - 涉及业务逻辑的判断条件

4. **模块顶部**：每个 `.py` 文件开头加模块级 docstring，说明该文件的职责。

5. **注释要解释 "为什么"，而不只是 "是什么"**：变量名已经能说明 "是什么"，注释应该补充意图、约束、坑点。

### 例外
- 测试文件中的样板代码（如 fixture 定义）若已足够直观，可以简化注释。
- 自动生成的代码不强制。

### 这条规则覆盖默认行为
> Claude Code 默认 "不写注释"。本项目里 **不适用** —— 在本工程中，请按上述规则写注释。
